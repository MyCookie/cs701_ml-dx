## 2. Core Concepts: A Primer for Beginners

### Structured vs. Unstructured Data
To an ML engineer, the difference is not just about how it "looks," but how it is **indexed** and **processed**.

**Structured Data**
- **Definition:** Data that fits into a fixed schema (like a spreadsheet or a SQL database). It consists of rows and columns.
- **Example:** A table of turbine temperatures where every entry has a `timestamp`, a `sensor_id`, and a `value`.
- **ML Processing:** These are often fed into "Classical" ML models (e.g., Random Forests, Linear Regression) as numerical features.

**Unstructured Data**
- **Definition:** Data that does not have a pre-defined model or is not organized in a pre-defined manner.
- **Example:** PDF reports of maintenance logs, X-ray images, or audio recordings of engine noise.
- **ML Processing:** These require "Feature Extraction" (e.g., using a CNN for images or a Transformer for text) to convert them into a mathematical format the model can understand.

> **Visual Aid Suggestion: The Data Funnel Diagram**
> *Imagine a diagram showing a wide mouth (Unstructured Data: PDFs, Images, Audio) being squeezed through a filter (Feature Extraction/Embedding) into a narrow pipe (Structured Vector/Tensor) that then enters the ML Model.*

### Real-world vs. Synthetic Data
**Real-world Data** is captured from actual physical events. It is the "Truth," but it is often expensive, rare, or private (e.g., patient records).

**Synthetic Data** is "fake" data generated by another algorithm (e.g., a GAN or an LLM) that mimics the statistical properties of real data.
- **Pros:** Infinite supply, no privacy concerns, can create "edge cases" that rarely happen in the real world.
- **Cons: Model Collapse.** If a model is trained *only* on synthetic data produced by another model, it can enter a feedback loop where it forgets the nuances of the real world and begins to amplify its own errors.

## 3. The Scenario: "Project Nexus"

Your organization is building a "Knowledge Assistant" for a complex engineering field (e.g., Aerospace Maintenance). You have access to the following data sources:
1. **Sensor Logs:** Time-series data of pressure and temperature.
2. **Maintenance Manuals:** 500-page PDFs containing technical instructions.
3. **Engineer Notes:** Handwritten notes scanned as images.
4. **Failure Reports:** Standardized forms with checkboxes and short text fields.

### Task 1: The Taxonomy Classification
Categorize the sources above. For each, explain *why* it is structured or unstructured and what the "Translation Challenge" would be for an ML engineer.

| Data Source | Taxonomy | Why? | Translation Challenge |
| :--- | :--- | :--- | :--- |
| Sensor Logs | **Structured** | Fixed columns (Time, Value). | Handling "gaps" in time-series. |
| Manuals | **Unstructured** | Free-form text in PDFs. | Converting PDF layout to linear text. |
| Engineer Notes | **Unstructured** | Image-based handwriting. | OCR (Optical Character Recognition) errors. |
| Failure Reports | **Semi-Structured** | Mix of checkboxes and text. | Mapping "Checkboxes" to binary (0/1). |

### Task 2: The Synthetic Data Strategy
You realize that you have very few examples of "Catastrophic Engine Failure" (only 5 cases in 10 years). You cannot train a model on 5 examples.

**Question:** How would you use synthetic data to solve this? 
**Constraint:** Describe how you would ensure the synthetic data is "Domain-Valid" (i.e., it doesn't describe a physical impossibility).

## 4. The Deliverable: The Data Strategy Map

Your final deliverable is a **Data Strategy Map**. This is a high-level document that guides the ML engineer on how to handle the project's data assets.

### Requirements for the Map:
1. **Inventory:** A list of all data sources and their taxonomy.
2. **Processing Path:** For each unstructured source, suggest a method to make it "model-ready" (e.g., "Use OCR for notes," "Use a PDF Parser for manuals").
3. **Synthetic Augmentation Plan:** 
    - Which classes of data need synthetic augmentation?
    - Who is the "Human-in-the-Loop" (the Expert) who will validate the synthetic data?
4. **Risk Assessment:** A brief paragraph on how you will prevent **Model Collapse**.
